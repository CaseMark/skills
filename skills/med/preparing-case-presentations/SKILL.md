# Preparing Case Presentations

Formats clinical cases for formal presentation settings.

## Workflow

1. **Determine** audience and format (morning report, tumor board, M&M, grand rounds)
2. **Select** relevant history — filter for presentation-appropriate detail
3. **Structure** by standard format (varies by setting)
4. **Embed** clinical reasoning at decision points
5. **Prepare** discussion points and questions

## Morning Report / Case Conference Format

```markdown
## Case Presentation

### One-Liner
[Age, sex, relevant PMH, presenting with chief complaint and key feature]

### HPI
[Narrative — focused, chronological, relevant negatives included]

### PMH/PSH
[Pertinent items only]

### Medications & Allergies
[Current list]

### Social/Family History
[If relevant]

### Physical Exam
[Pertinent positives and negatives]

### Diagnostics
[Key labs, imaging, pathology]

### Assessment
[Problem-based assessment with clinical reasoning]

### Plan
[By problem]

### Discussion Points
- [Teaching point or clinical question]
```

## Key Rules

- One-liner sets the frame — make it count
- Include relevant negatives that narrow the differential
- For teaching cases, embed reasoning ("this finding makes X more likely because...")
- Tumor board presentations must include pathology and imaging details
- M&M format requires event timeline and systems analysis
- Adjust detail level to audience (residents vs. attendings vs. multidisciplinary)
