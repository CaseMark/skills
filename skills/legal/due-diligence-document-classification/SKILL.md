# Due Diligence Document Classification

Classifies, organizes, and indexes data room documents to accelerate due diligence review and identify documentation gaps.

## Prerequisites

- Access to data room or document repository
- Due diligence checklist or request list
- Document naming conventions (if any exist)
- Transaction type (M&A, investment, audit, real estate)

## Workflow

### 1. Ingest and Parse Documents

Process every document in the data room. Extract: document type, date, parties, subject matter, and key terms. Handle multiple formats (PDF, Word, scanned images with OCR).

### 2. Classify by Category

Assign each document to a category: corporate governance, financial statements, material contracts, intellectual property, employment, litigation, real property, regulatory filings, insurance, or tax. Sub-classify within categories (e.g., contracts → customer/vendor/employment/lease).

### 3. Assess Relevance and Risk

Score each document: critical (directly affects deal terms or valuation), important (relevant to ongoing obligations), reference (background context), or duplicate/superseded. Flag documents containing red-flag language: pending litigation, environmental liability, change of control restrictions, consent requirements.

### 4. Map Against Request List

Cross-reference classified documents against the due diligence request list. Identify: fully satisfied requests, partially satisfied (documents exist but are incomplete), and unsatisfied (no responsive documents found).

### 5. Produce Index and Gap Report

Generate a structured index with document ID, classification, relevance score, and location. Append a gap analysis listing every unfulfilled request with urgency level and suggested follow-up.

## Pitfalls

- Treating document titles as reliable indicators of content (mislabeled files are common)
- Missing documents embedded as exhibits or attachments within other documents
- Not flagging documents that are responsive to multiple request categories
