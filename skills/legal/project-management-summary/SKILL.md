# Legal Project Management Summary

Deliver a concise, decision-ready snapshot of a legal matter’s status, plan, and risks.

## Prerequisites

1. Matter file summary or project charter
2. Latest task list or workplan
3. Budget baseline and current spend
4. Key dates and deadlines
5. Team roster and roles
6. Recent developments or decisions
7. Audience and distribution scope

## Output Structure / Process

Use this structure and fill every section.

**Header**
- Matter name:
- Client:
- Lead counsel:
- Prepared by:
- As‑of date:
- Distribution level: `Internal` or `Client` or `Restricted`

**1) Executive Overview (2–3 sentences)**
- Current status and primary driver
- Immediate attention items

**2) Objectives and Scope**
| Objective | Business/Legal Driver | Deliverable | Owner | Status |
|---|---|---|---|---|

**3) Timeline**
| Milestone | Target Date | Actual Date | Dependency | Owner | Status |
|---|---|---|---|---|---|

**4) Budget**
| Budget Item | Baseline | Spent To Date | Forecast To Complete | Variance | Driver |
|---|---|---|---|---|---|

**5) Resources**
| Name/Role | Allocation (%) | Key Responsibilities | Notes |
|---|---|---|---|

**6) Status Assessment**
| Dimension | R/Y/G | Evidence | Corrective Action |
|---|---|---|---|
| Schedule |  |  |  |
| Budget |  |  |  |
| Quality |  |  |  |
| Team |  |  |  |
| Client |  |  |  |
| Risk |  |  |  |

**7) Risks and Issues**
| Risk/Issue | Impact | Likelihood | Mitigation | Owner | Due Date |
|---|---|---|---|---|---|

**8) Decisions and Approvals**
| Decision Needed | Options | Recommendation | Decision Maker | Due Date |
|---|---|---|---|---|

**9) Action Items**
| Action | Owner | Due Date | Success Criteria |
|---|---|---|---|

**10) Source References**
- List documents, emails, reports, and data sources used.

## Guidelines

- Mark privileged or work‑product sections clearly as `PRIVILEGED/WORK PRODUCT`.
- If the audience is external, omit attorney mental impressions and litigation strategy.
- Use dates in `YYYY-MM-DD` and name responsible parties consistently.
- Highlight root causes for variances and specify corrective owners and deadlines.
- Keep legal analysis minimal unless the audience is internal.
- Note jurisdictional constraints or court deadlines explicitly when applicable.
