# C-TPAT Security Profile

Produce an evidence-based, internally consistent C-TPAT profile draft aligned to current U.S. CBP expectations and directly mapped to company operations.

## Prerequisites

1. Governing documents: security policies, SOPs, training materials, incident logs, internal audits, partner questionnaires.
2. Corporate identifiers: legal name, EIN/Federal Tax ID, DUNS (if any), C-TPAT account number (if enrolled), facility names/addresses.
3. Operations data: importer role, import volumes, countries/lanes, suppliers, logistics chain map.
4. Security evidence for each module: physical controls, personnel screening/training, cargo handling, conveyance checks, IT controls, business partner records.
5. Document version/date baseline and date-of-submission target.

## Output Structure / Process

1. Validate inputs for specificity and verify each factual claim to source documents.
2. If required facts are missing, pause and emit a `Gap Register` before drafting narrative sections.
3. Populate profile using the template below, then run internal consistency checks across sections.
4. Add compliance crosswalks: company control → C-TPAT minimums → supporting evidence.
5. Finish with quality checks: accuracy, contradiction scan, completeness by section, tone appropriate for CBP/management audience.

### Required Intake Matrix

| Item | Required Evidence | Output Treatment |
|---|---|---|
| Company eligibility profile | Import role, volume/frequency, facilities, prior tier status | Intro + jurisdiction statement |
| Governance | Security leadership, reporting lines, budget authority | Org/security oversight section |
| Risk methodology | Assessment cadence, risk criteria, data sources | Risk framework section |
| Physical controls | Access control, surveillance, lighting, alarms, patrols | Physical security section |
| Personnel security | Screening, onboarding, training, revocations | Personnel section |
| Procedural controls | Partner due diligence, custody chain, inspections, records | Procedural security section |
| Conveyance security | Carrier vetting, 7-point inspections, seals, transit controls | Transportation section |
| IT security | MFA, segmentation, patching, incident response | IT resilience section |
| Compliance program | Self-assessments, corrective actions, validation prep | Continuous improvement section |

### Mandatory Draft Sections

1. Cover Page
   - Company information, document title, version, date, prepared by.
2. Profile Summary
   - Program tier objective, participation status, confidence level.
3. Organizational Framework
4. Risk Assessment
5. Physical Security
6. Personnel Security
7. Procedural/Business-Partner Security
8. Conveyance and Cargo Integrity
9. Information Technology Security
10. Recordkeeping and Compliance Management
11. Self-Assessment and Corrective Action Register
12. Validation Readiness
13. Appendix Index and Signature Statement

## Guidelines

1. Use only verifiable facts from provided records; avoid aspirational language.
2. For each claim, include a source pointer (policy, SOP, log, audit report, contract, training record).
3. Use exact terms from company records for titles, systems, equipment models, and locations.
4. Keep language factual and concise; no legal boilerplate where evidence controls.
5. Include [VERIFY] where regulatory specifics are uncertain or require current source confirmation.
6. U.S.-only scope unless user provides explicit multinational authority requirements; default to CBP program expectations for C-TPAT [VERIFY].
7. Never invent documents, certifications, or findings.
8. Enforce 7-point inspection checks and ISO 17712-seal handling for high-security seals.
9. Ensure document is ready for validator review: consistency, indexed exhibits, and revision control.
