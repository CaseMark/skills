# Client Advisory Summary

Produces a professional advisory memorandum translating legal developments into tailored client guidance with impact analysis and actionable recommendations.

## Prerequisites

1. **Legal development(s)** — case law, statutory amendments, regulatory updates, administrative guidance, or pending legislation to cover
2. **Client profile** — industry, jurisdiction(s), business operations, and specific legal concerns
3. **Client materials** (if available) — contracts, policies, prior advisories, compliance frameworks that may be affected
4. **Effective dates and deadlines** — any time-sensitive compliance windows

## Output Structure

### 1. Header

| Field | Content |
|-------|---------|
| To | Client name / matter ID |
| From | Attorney / firm |
| Date | Date of preparation |
| Re | Concise subject line identifying development(s) |

### 2. Executive Summary (2–3 paragraphs)

- Lead with the most critical development and its immediate implication
- Write for a non-lawyer audience while preserving legal precision
- State the bottom line: what changed, who is affected, what to do

### 3. Body Sections (one per development or topic area)

For each development, use this framework:

| Component | Content |
|-----------|---------|
| **Development** | What changed — statute, rule, case holding, agency action |
| **Citation** | Full Bluebook citation; mark unverified cites with `[VERIFY]` |
| **Effective Date** | When it takes effect or applies |
| **Context** | Brief origin/purpose (1–2 sentences max) |
| **Substantive Changes** | Bullet list of new requirements, standards, or rights |
| **Client Impact** | How it affects this client's operations, obligations, or rights |
| **Impact Horizon** | Immediate / Short-term / Long-term |
| **Impact Type** | Compliance obligation / Strategic opportunity / Risk exposure |
| **Recommendations** | Numbered action items with deadlines where applicable |

### 4. Recommendations Summary Table

| # | Action Item | Deadline | Priority | Type |
|---|------------|----------|----------|------|
| 1 | ... | ... | High/Med/Low | Compliance/Strategic/Risk |

### 5. Closing

- Note any assumptions or limitations in the analysis
- Flag areas requiring further fact-gathering or follow-up
- Invite client discussion with specific contact information

## Drafting Checklist

- [ ] Every factual assertion tied to a cited source
- [ ] All citations in Bluebook format; uncertain cites marked `[VERIFY]`
- [ ] Cross-referenced client's existing contracts/policies against new requirements
- [ ] Jargon defined on first use or replaced with plain language
- [ ] Concrete examples illustrating impact on client's actual operations
- [ ] All deadlines and effective dates explicitly stated
- [ ] Recommendations are specific and actionable (not "consider reviewing")

## Guidelines

- **Length**: 2–5 pages depending on complexity; bias toward concision
- **Tone**: Professional, client-focused, accessible — not academic
- **Visual aids**: Use comparison tables for before/after regulatory changes, timelines for phased implementation, flowcharts for decision points
- **Citations**: Bluebook format throughout; use footnotes/endnotes for detailed citations, keep main text clean
- **Limitations**: If analysis depends on unconfirmed client facts, state assumptions explicitly and recommend verification
- **Jurisdiction**: Default US; flag state-specific variations where federal and state law diverge
- **No speculation**: Distinguish between enacted law and proposed/pending changes; label each clearly
