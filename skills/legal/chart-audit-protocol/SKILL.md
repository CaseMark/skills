# Chart Audit Protocol

Drafts a defensible chart audit protocol for healthcare organizations that functions as both an operational roadmap and a regulatory compliance document.

## Prerequisites

1. **Audit trigger and scope** — routine periodic, targeted risk review, RAC preparedness, or post-regulatory-update
2. **Regulatory driver** — specify: Medicare CoPs, OIG guidance, payer contract, state licensing, or internal compliance program
3. **Scope parameters** — timeframe, departments/providers, service types, patient populations
4. **Sampling approach** — random, stratified, or targeted; universe size and desired confidence level
5. **Prior audit findings** — benchmarks, historical error rates, identified risk areas

## Output Structure

### 1. Purpose Statement

| Element | Content |
|---|---|
| Regulatory framework | Cite specific driver (Medicare CoPs, OIG CPG, RAC, payer contract, state requirement) |
| Audit classification | Routine / targeted risk / proactive post-regulatory |
| Integration rationale | How this audit fulfills the duty to monitor under federal sentencing guidelines and OIG guidance |
| Governance alignment | Compliance committee / board oversight connection |

---

### 2. Scope Definition

| Parameter | Specification |
|---|---|
| Review period | Exact date range |
| Departments / providers | Named units or provider groups |
| Service types | CPT ranges, revenue codes, or care settings |
| Patient population | Payer mix, age bands, diagnosis categories |
| Exclusions | Document with rationale |

**Sampling Methodology Block**

- [ ] Simple random sampling
- [ ] Stratified sampling (risk-factor tiers: high / medium / low)
- [ ] Targeted selection (data analytics, prior findings, denial patterns)

Document: universe size · sample size · confidence level (typically 90–95%) · margin of error · extrapolation basis for overpayment calculations

---

### 3. Documentation Review Criteria

**Required per-record elements:**

- [ ] Patient identification on each page
- [ ] Date and time of service
- [ ] Chief complaint / reason for encounter
- [ ] HPI (detail appropriate to E&M level)
- [ ] ROS (when applicable to service)
- [ ] PMH / surgical history / family and social history
- [ ] Current medications and allergies
- [ ] Physical examination findings
- [ ] Assessment (reflects clinical judgment)
- [ ] Treatment plan with follow-up instructions
- [ ] Legible, authenticated, and complete entries

**Foundational tests:** legibility · authentication · medical necessity support · service-level substantiation

---

### 4. Coding Accuracy Assessment

| Review Area | Key Questions |
|---|---|
| CPT / HCPCS accuracy | Do reported codes match documented procedures? |
| E&M level support | Is level supported by history + exam + MDM complexity under current AMA guidelines? |
| Diagnosis coding | Are ICD codes clinically supported, correctly sequenced, principal dx = primary reason for encounter? |
| Modifier usage | Are modifiers (bilateral, distinct service, multiple physician) supported by documentation? |
| NCCI compliance | Any inappropriate unbundling? Overrides supported by documentation of distinct services? |

---

### 5. Regulatory Compliance Checklist

- [ ] Provider credentialing and current privileges for procedures performed
- [ ] Supervision requirements met for residents, PAs, NPs, other NPPs
- [ ] Incident-to billing requirements satisfied (when applicable)
- [ ] Rendering provider correctly identified on claim
- [ ] Shared/split visit billing complies with current Medicare and payer policy
- [ ] Frequency limitations and LCD/NCD coverage determinations observed
- [ ] ABN issued and documented where coverage was uncertain

---

### 6. Findings Report Structure

```
1. Executive Summary
   - Overall error rate
   - Estimated financial exposure
   - Top 3 systemic issues requiring immediate action

2. Methodology
   - Sampling design, universe/sample sizes, confidence level
   - Reviewer qualifications and independence
   - Review criteria and coding guidelines applied
   - Limitations and assumptions

3. Quantitative Findings (by category)
   - Documentation deficiencies (no payment impact)
   - Coding errors → overpayment
   - Coding errors → underpayment
   - Compliance violations (regulatory risk, payment-neutral)
   - Extrapolated overpayment with confidence interval (if statistically valid)
   - Trend comparison to prior audits / industry benchmarks

4. Risk Categorization
   - Technical / low risk: minor omissions, no medical necessity impact
   - Substantive / high risk: upcoding, services not rendered, medically unnecessary procedures

5. Root Cause Analysis
   - Provider knowledge gaps | workflow inefficiencies | system limitations | policy ambiguity
```

---

### 7. Corrective Action Plan

For each finding category:

| Element | Detail |
|---|---|
| Remediation action | Provider education / pre-bill review / CDI program / system change / policy update |
| Responsible party | Named individual or department |
| Implementation deadline | Specific date |
| Success metric | Target error rate / benchmark |
| Follow-up audit date | Re-audit scope and timing |

---

### 8. Self-Disclosure and Overpayment Analysis

- [ ] Do identified overpayments trigger mandatory 60-day refund obligation? (42 U.S.C. § 1320a-7k(d)) [VERIFY current CMS guidance on identification date]
- [ ] Do error patterns suggest systematic issues warranting OIG Self-Disclosure Protocol submission?
- [ ] Quantify overpayment and document refund/offset approach
- [ ] Stakeholder communication plan: individual providers · department leaders · compliance committee · board

---

## Guidelines

- **Privilege**: If conducted under attorney direction, document privilege basis; assume all records may be discoverable in government investigations or litigation
- **Language discipline**: Avoid admissions of intent; frame findings as compliance improvement opportunities
- **Retention**: Maintain per applicable federal requirements and organizational compliance program policy
- **Extrapolation**: Only project overpayments statistically when sampling is properly designed; document methodology to withstand RAC or DOJ scrutiny
- **Confidentiality**: Do not identify individual patients or providers in ways creating HIPAA exposure in widely distributed reports
- **Jurisdiction**: US federal framework (Medicare/Medicaid); verify applicable state requirements for Medicaid-specific audits
