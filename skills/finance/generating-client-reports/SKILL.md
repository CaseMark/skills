# Generating Client Reports

Produces client-appropriate portfolio communications.

## Workflow

1. **Summarize** performance — returns vs. benchmark, key contributors/detractors
2. **Contextualize** — market environment, economic backdrop
3. **Review** allocation — current vs. target, rebalancing actions
4. **Explain** activity — trades made and rationale
5. **Outlook** — forward view and positioning
6. **Adjust** tone for client type (institutional vs. retail, sophisticated vs. general)

## Output Template

```markdown
## Portfolio Review: [Client/Account] — [Period]

### Performance Snapshot
| Metric | Portfolio | Benchmark |
|--------|-----------|-----------|
| Return (QTD) | | |
| Return (YTD) | | |
| Return (1-Year) | | |

### Market Commentary
[2-3 paragraph overview of market environment during the period]

### Portfolio Activity
| Action | Security | Rationale |
|--------|----------|-----------|
| Bought | | |
| Sold | | |
| Rebalanced | | |

### Asset Allocation
| Asset Class | Current % | Target % | Over/Under |
|------------|-----------|----------|------------|

### Top Contributors / Detractors
| Holding | Contribution |
|---------|-------------|

### Outlook & Positioning
[Forward-looking view — risks being managed, opportunities being pursued]

### Disclosures
[Required compliance disclosures]
```

## Key Rules

- Retail clients: plain language, focus on goals, avoid jargon
- Institutional clients: more technical detail, factor attribution, benchmark-aware
- Performance must be GIPS-compliant if the firm claims GIPS compliance
- Always include required disclosures (varies by jurisdiction and firm)
- Explain underperformance honestly — don't hide behind market commentary
- Forward-looking statements need appropriate disclaimers
